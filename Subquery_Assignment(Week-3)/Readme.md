# WEEK-3 – Subquery & Window Functions (Superstore SQL Analysis)

## 📌 Project Overview

This project focuses on analyzing a Superstore dataset using advanced SQL concepts such as subqueries, Common Table Expressions (CTEs), joins, and window functions. The raw transactional data is transformed into a structured relational format to extract meaningful business insights.

The main goal is to understand customer purchasing behavior, identify high-value customers, and analyze sales distribution using analytical SQL techniques.

---

## 🎯 Objective

The objectives of this project are:

- Transform raw sales data into a structured relational database
- Perform customer-level and order-level analysis
- Use subqueries to compare individual records with aggregated results
- Apply CTEs for cleaner and modular query design
- Use window functions for ranking and segmentation
- Identify top-performing and low-performing customers
- Generate actionable business insights from sales data

---

## 🗄️ Database Design Overview

The database is structured into three core entities:

### 👤 Customers Table
Stores unique customer information such as customer ID, name, segment, and geographical details. This helps in analyzing customer distribution and segmentation.

### 📦 Products Table
Contains product-level details including category, sub-category, and product name. This supports product performance and category-based analysis.

### 📑 Orders Table
Represents transactional data including sales, quantity, discount, and profit. This is the central table used for all analytical queries.

---

## 🚀 Project Workflow

### 1️⃣ Data Setup & Understanding
The project begins by loading raw data and understanding its structure. Initial checks ensure data completeness and consistency before transformation.

---

### 2️⃣ Data Modeling & Normalization
Raw data is split into logical tables to remove redundancy and improve data integrity. Customers and products are separated from order transactions to create a clean relational structure.

---

### 3️⃣ Data Cleaning Approach
Duplicate records are handled during insertion by selecting distinct or grouped values. This ensures each customer and product is uniquely represented in the database.

---

## 📊 Analytical Approach

### 🔍 Subquery-Based Analysis

Subqueries are used to compare individual records against aggregated values. This helps in identifying:

- Orders that are higher than the average sales value
- Customers with exceptionally high-value transactions
- Records that stand out compared to overall dataset behavior

This approach provides a dynamic way to benchmark individual performance against the entire dataset.

---

### 📌 Customer-Level Aggregation (CTE Approach)

Common Table Expressions are used to calculate total sales per customer in a structured and reusable format. This simplifies complex queries and improves readability.

Using CTEs, we can easily analyze:

- Total revenue generated by each customer
- Customers contributing above-average sales
- Distribution of sales across the customer base

---

### 🏆 Ranking & Segmentation (Window Functions)

Window functions are applied to rank customers based on their total sales without collapsing row-level data.

This enables:

- Ranking customers from highest to lowest contribution
- Assigning row numbers to individual customer orders
- Identifying top-performing customers efficiently

This approach is crucial for business decision-making and customer prioritization.

### 🧠 Behavioral Insights

- Some customers generate high revenue through fewer but larger orders
